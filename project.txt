Grid of cards
A Grid of cards is a grid-like layout for displaying multiple cards.

One simple way to display our top albums data is just create a list of all the albums.

But, we want to provide a good UI for our users at QTify. So, instead of a normal list, we can make use of cards to display the information about a single album, and to display data about multiple such albums we can make use of a Grid layout to display multiple cards resulting in a good UI.

image

If you notice in the given Figma design, you’ll see we have multiple sections wherein we have grids: Top Albums and New Albums (clicking on that Show All for New Albums will show a Grid for New Albums, just like it does for Top Albums).

So, we can make a reusable component called a Section for implementing the Grid.

Implementation Requirements for Section:
Create a reusable component called a Section.

Using Axios, fetch data for top albums from the following API endpoint:

https://qtify-backend.labs.crio.do/albums/top

Make use of the reusable Card component to include multiple cards in the Top Albums grid.

To create the grid layout, make use of the CSS Flexbox or Grid.

Make sure that there’s proper horizontal spacing between the Top Albums text and the Collapse button. Also, ensure there is proper vertical spacing between the Top Albums text and the Grid, as shown in the Figma design.

Carousel
Carousel allows the users to view a group of content (in our case, Album Cards) from start to end or even cycle through it(i.e. end is again connected to the start).

For our QTify, we would be building a Carousel that lets the users view a group of Album cards at a time, and then slide to the left or right to view more cards. We won’t make it cyclic, so once the user reaches the last card, the right navigation button will vanish. Similarly, we will build for the left navigation as well.

Implementation Requirements for Top Albums Carousel:
Create a separate component called Carousel.

Conditional Rendering: upon clicking the Collapse button:

The text within the button must change to "Show All"

Instead of a Grid of Cards, the Carousel component must be displayed.

Make use of the Swiper open-source UI library to build the Carousel.

Make sure that you make the Carousel component re-usable i.e. it should be built in such a way that if in the future you are required to change the content from cards to some other component or even text, you should be able to do so without making any changes to the carousel except of passing the new data in the form of props.

Make sure that the Carousel is responsive i.e. the number of cards getting displayed should vary as per the screen size appropriately and not just show a fixed number of cards no matter what the size is.

Export the Left and Right navigation buttons SVG from the Figma file.

Once you have a well-functioning Carousel, include the left and right navigation buttons in the carousel. Create separate components for the left and right navigation buttons.
